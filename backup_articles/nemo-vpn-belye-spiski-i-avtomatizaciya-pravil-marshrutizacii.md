  ## Что такое белые списки

Белый список IP-адресов — это база, которую ведут платформы совместно с регулятором. Если ваш IP в этом списке — вам разрешён доступ. Если нет — вас могут ограничить или заблокировать.

Кому раздали белые списки:
      - Маркетплейсы: Wildberries, Ozon
      - Поисковик: Яндекс
      - Соцсети: VK
      - Банки: Сбербанк, ВТБ, Тинькофф и другие
      - Другие крупные сервисы

Проблема в том, что белые списки включают в себя:
      - IP российских провайдеров (МТС, Билайн, Tele2, МегаФон)
      - IP облачных провайдеров (Яндекс.Облако, Selectel, Селектел)
      - Но НЕ включают IP дата-центров за рубежом (Hetzner, DigitalOcean, AWS, Vultr)

  

Если ваш VPN использует IP дата-центра за рубежом — вы в списке «подозрительных».

На iPhone iOS есть одно важное ограничение: VPN-конфигурация не может быть динамической. Если вы включили VPN — он работает для всех приложений. Отключили — не работает ни для одного.

NEMO VPN решает это с помощью инструкции, которая автоматически настраивает VPN-профиль для iOS:

1. Вы получаете в боте конфигурацию для iOS
2. Устанавливаете профиль
3. Профиль содержит правила маршрутизации
4. При открытии приложения РУ компании VPN автоматически включается, при закрытии — отключается

Как это работает технически:
      - Профиль использует функцию Per-App VPN в iOS 17+
      - Список доменов для прямого доступа
      - Список доменов для VPN-доступа
      - Автоматическое переключение

  

Вам не нужно каждый раз включать/отключать VPN. Всё происходит автоматически.

  ### 4. Резидентский IP (residential IP)

  

Большинство VPN-сервисов используют IP дата-центров за рубежом. Эти IP-адреса внесены в черные списки платформ, потому что они хорошо известны: Hetzner AS24940, DigitalOcean AS14061, AWS AS16509.

NEMO VPN использует резидентские IP-адреса российских провайдеров. Эти IP:
      - В белых списках платформ (потому что это российские провайдеры)
      - Не выглядят как дата-центры (потому что это обычные пользовательские адреса)
      - Стабильно работают с российскими сервисами

  

Но при этом для заблокированных ресурсов используется защита — ваш реальный IP скрыт, и платформа видит IP российского провайдера, а не ваш реальный адрес.

  ## Почему другие VPN не делают этого

Автоматизация маршрутизации требует:
1. Разработки собственного клиента
2. Интеграции с платформами (отслеживание белых списков)
3. Обновления правил при изменениях (белые списки постоянно меняются)

Большинство VPN-сервисов этого не делают. Им проще:
