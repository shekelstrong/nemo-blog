# Что происходит с VPN в России: апрель 2026 — ТСПУ, белые списки, оплата трафика и что работает

**Апрель 2026** — это месяц, когда всё изменилось. Обычный VPN умер. DPI научился детектировать WireGuard, AmneziaWG, Cloak и даже XRay VLESS. Белые списки платформ — 57 000 адресов для 1730 предприятий — блокируют пользователей с VPN. С 1 мая вводится плата за международный трафик: 15 ГБ бесплатно, далее ~150 рублей за гигабайт.

В этой статье разберём: что именно произошло за последние 3 недели, какие протоколы перестали работать, почему XHTTP важнее TLS, чем отличается Hysteria2 от WireGuard и что делать IT-специалистам, которые теперь тратят 30% рабочего времени на борьбу с DPI.

## Апрель 2026: итоги — что изменилось за последние 3 недели

### Масштабная эскалация блокировок

Период **начало апреля – конец апреля 2026** стал катастрофическим для VPN-пользователей в России:

- **Дни 5–10 апреля**: ТСПУ развернули новую конфигурацию DPI — массовые падения WireGuard и OpenVPN
- **Дни 11–15 апреля**: Обнаружена детекция AmneziaWG, Cloak и стандартных конфигураций XRay VLESS
- **Дни 18–19 апреля (выходные)**: Массовые перебои XRay VLESS по всей России — провайдеры МегаФон и другие крупные игроки начали активно блокировать Reality
- **День 15 апреля**: Платформы (Ozon, WB, Яндекс, VK, банки) начали блокировать VPN-пользователей на уровне белых списков

### Что сломалось за 3 недели

До **начало апреля** работали:

- WireGuard — в базовой конфигурации
- OpenVPN — стандартный порт 1194/443
- AmneziaWG — "свободный" режим без обфускации
- XRay VLESS — базовая Reality с дефолтным Server Name

После **конец апреля**:

- **WireGuard** — детектируется по заголовкам пакетов (DPI видит характерный паттерн)
- **OpenVPN** — блокируется на этапе handshake (TLS-сигнатуры известны)
- **AmneziaWG** — перестала работать в стандартных настройках
- **XRay VLESS** — Reality блокируется в выходные, требует постоянной ротации fingerprint

### Важное отличие: массовые vs избирательные блокировки

ТСПУ 2.0 переключилась от **массовых блокировок IP-адресов** к **избирательному детектированию протоколов**:

| Режим | Раньше (до апреля 2026) | Сейчас (апрель 2026) |
|-------|------------------------|----------------------|
| **Метод** | Блокировка IP-адресов VPN-серверов | DPI-анализ TLS handshake и структуры пакетов |
| **Что работает** | Смена IP-адреса сработает | Смена IP бесполезна — блокируется протокол |
| **Пример** | IP 123.45.67.89 в реестре → трафик режется | Любой IP с WireGuard handshake → трафик режется |
| **Обход** | Резервный сервер в другой стране | Изменение протокола (Reality → XHTTP) |

## ТСПУ 2.0: новые методы блокировки VPN-трафика

### Что такое ТСПУ и как он работает

**ТСПУ** (Технические средства противодействия угрозам) — система DPI (Deep Packet Inspection), которую провайдеры обязаны устанавливать по закону №278-ФЗ (2017). ТСПУ:

- Анализирует **каждый пакет** трафика в реальном времени
- Ищет **характерные сигнатуры** протоколов (TLS, HTTP, VPN)
- Блокирует, режет или замедляет трафик

До 2026 года ТСПУ использовал **статические правила** — реестры IP-адресов и доменов. В апреле 2026 включили **динамический анализ**:

- TLS fingerprinting (JA3/JA4 хэши)
- Active probing (активное зондирование серверов)
- ML-модели для детекции аномалий в поведении клиентов

### DPI теперь видит протоколы по заголовкам

**WireGuard** считался "невидимым" для DPI, потому что:

- Зашифрован с использованием Noise Protocol
- Нет handshake в привычном смысле (всё в одном пакете)
- Сервер и клиент равноправны

**Но в апреле 2026 ТСПУ научился детектировать WireGuard:**

```go
// Упрощённый пример того, что видит DPI
if packet[0..4] == []byte{0x01, 0x00, 0x00, 0x00} {  // Тип сообщения = Handshake Initiation
    if packet[4] == 0x12 {  // Резервный байт
        if contains_noise_pattern(packet[8:]) {
            block_connection()  // Это WireGuard!
        }
    }
}
```

ТСПУ ищет:

1. **Первый байт** = 0x01 (Handshake Initiation)
2. **Резервный байт** = 0x12 (константа WireGuard)
3. **Шум в начале** — случайные данные для скрытия ключей

Этого достаточно для уверенной детекции.

### OpenVPN блокируется по TLS handshake

**OpenVPN** использует TLS 1.2/1.3 для защиты handshake. ТСПУ видит:

- TLS ClientHello с SNI (имя сервера)
- JA3 хэш — уникальный отпечаток клиентского TLS-implementation
- User-Agent или характерный pattern в payload

Даже если OpenVPN работает на порту 443:

```
TLS ClientHello → JA3 hash → DPI сравнивает с реестром OpenVPN fingerprints → БЛОК
```

### XRay VLESS: Reality и её слабости

**VLESS Reality** — протокол для XRay, который имитирует обычный HTTPS-сайт:

- Подменяет TLS handshake — выглядит как посещение legit-домена (например, google.com)
- SNI = dest (адрес, куда реально идёт трафик)
- Server Name = легитимный домен (например, www.microsoft.com)

**В апреле 2026 Reality начали блокировать:**

1. **Пятница 18 апреля** — первые перебои на МегаФоне
2. **Суббота 19 апреля** — массовые жалобы на невозможность подключиться
3. **Воскресенье 20 апреля** — провайдеры начали активно режет Reality по всей России

**Почему Reality сломалась:**

- **Fingerprint** — большинство конфигураций использует `fingerprint: "chrome"`, который легко детектируется
- **Server Name** — если 1000+ пользователей используют одинаковый Server Name (например, `www.google.com`), DPI видит аномалию
- **Active probing** — ТСПУ сама подключается к серверу, проверяет, отвечает ли он как legit-сайт

## Что перестало работать: WireGuard, OpenVPN, стандартный VLESS

### WireGuard: миф о невидимости развеян

**До апреля 2026:**

- WireGuard считался "невидимым" для DPI
- Блогеры писали "поставьте WireGuard — ТСПУ не увидит"
- VPN-провайдеры рекламировали WireGuard как будущее

**После апрельского обновления ТСПУ:**

- **WireGuard детектируется за 1 пакет** — достаточно handshake initiation
- **Смена порта не помогает** — структура пакета неизменна
- **Обфускация не работает** — нет стандартной поддержки

**Реальность:**

```
WireGuard (port 51820) → DPI видит handshake initiation → БЛОК
WireGuard (port 443)  → DPI видит handshake initiation → БЛОК
WireGuard (port 80)   → DPI видит handshake initiation → БЛОК
```

Любой порт — результат один и тот же.

### OpenVPN: классика, которая больше не работает

**OpenVPN** — самый старый из популярных VPN-протоколов:

- Появился в 2001 году
- Использует OpenSSL для шифрования
- Поддерживается всеми VPN-провайдерами

**Почему перестал работать:**

1. **TLS ClientHello** — видит ТСПУ (параметры cipher suites, extensions)
2. **JA3 fingerprint** — уникальный хэш для OpenVPN-клиента
3. **User-Agent** — иногда передаётся в HTTP-over-TLS

Даже если переключиться на **TCP-режим** и **порт 443**:

```
OpenVPN TCP/443 → DPI видит JA3 hash для OpenVPN → БЛОК
```

### VLESS Reality: работает только в выходные

**VLESS Reality** — самый популярный протокол для обхода блокировок в 2025–2026:

- Имитирует HTTPS-трафик
- Работает на порте 443
- Подменяет TLS certificate

**В апреле 2026 Reality работает:**

- **Понедельник – пятница** — более-менее стабильно
- **Суббота – воскресенье** — массовые перебои, провайдеры активнее блокируют

**Жалобы пользователей (выходные 18–20 апреля):**

- "Ничего не работает, даже Reality"
- "МегаФон режет всё"
- "Переключился на мобильный интернет — работает 5 минут, потом падает"
- "Приходится постоянно менять конфиг"

**Причина:** провайдеры активнее используют DPI в выходные, когда нагрузка ниже.

## Белые списки: 57 000 адресов, 1730 предприятий — как это работает

### Что такое белые списки платформ

**Белые списки** (whitelists) — это базы IP-адресов, которые платформы разрешают к доступу. Если ваш IP не в белом списке — вас блокируют.

**По данным Habr "30% рабочего времени — борьба с DPI":**

- **57 000 адресов** в белых списках
- **1730 предприятий** получили доступ
- Это включает банки, маркетплейсы, мессенджеры, соцсети

### Как работает блокировка по белым спискам

**Сценарий 1: вы пытаетесь зайти на Ozon с VPN**

```
1. Ваш IP: 185.xxx.xxx.xxx (VPN-сервер в Нидерландах)
2. Ozon проверяет IP → нет в белом списке
3. Ozon показывает CAPTCHA или блокирует страницу
```

**Сценарий 2: вы пытаетесь зайти на Ozon без VPN**

```
1. Ваш IP: 94.xxx.xxx.xxx (домашний интернет от Билайн)
2. Ozon проверяет IP → есть в белом списке (адрес Билайна)
3. Ozon пропускает вас
```

### Белые списки против residential IP

**Datacenter IP:**

- Адреса, принадлежащие VPS-провайдерам (DigitalOcean, Hetzner, Vultr)
- Сразу детектируются платформами
- Блокируются автоматически

**Residential IP:**

- Адреса, принадлежащие домашним ISP (AT&T, Verizon, Orange)
- Выглядят как обычный пользователь
- Менее вероятно, что заблокируют

**NEMO VPN** использует residential IP + VLESS Reality + XHTTP для обхода белых списков.

### Как платформы определяют VPN

**Методы детекции:**

1. **IP-адрес** — сравнение с базами datacenter IP (MaxMind, ipinfo.io)
2. **ASN** — автономная система (например, Hetzner Online GmbH)
3. **Направление трафика** — если вы запрашиваете много разных доменов в короткое время
4. **WebRTC** — утечка реального IP через WebRTC
5. **User-Agent** — несоответствие между платформой и User-Agent

**Почему iPhone сложнее детектировать:**

- Apple Network Extension — нет доступа к WebRTC
- Safari и Chrome изолируют WebRTC по умолчанию
- Аппаратные фичи (Secure Enclave)

## 15 ГБ бесплатно, далее 150 руб/ГБ: «налог на зарубежный трафик»

### Что это за плата и когда вводится

С **1 мая 2026** вводится плата за международный трафик:

- **Первые 15 ГБ** — бесплатно
- **Каждый гигабайт сверх** — ~150 рублей
- **Кто платит** — провайдеры, но расходы лягут на пользователей

### Как это работает технически

**Схема:**

```
Ваш телефон → 4G LTE → Провайдер (МТС) → Шлюз → Интернет
                        ↓
                  DPI считает трафик
                  на зарубежные ресурсы
                        ↓
           Свыше 15 ГБ → +150 руб/ГБ к счёту
```

**Что считается "международным трафиком":**

- Трафик на серверы за пределами России (IP не в диапазонах RIPE NCC для РФ)
- Сайты, зарегистрированные на зарубежные домены (.com, .org, .net)
- VPN-трафик (туннель заканчивается за границей)

**Что НЕ считается:**

- Российские сайты (.ru, .рф)
- Сайты, зарегистрированные на домены .su
- Серверы в России (VK, Яндекс, Ozon, WB)

### Как обходить лимит 15 ГБ

**Способ 1: VPN на роутере**

Если вы настроите VPN на роутере с прошивкой OpenWRT:

- Домашний интернет не учитывается как мобильный
- Провайдеры считают только 4G/5G трафик
- Вы можете использовать неограниченный VPN-трафик через домашний интернет

**Способ 2: split tunneling**

Раздельный маршрутизатор:

- Российские сайты → напрямую (не расходует лимит)
- Зарубежные сайты → через VPN (расходует лимит)

**Способ 3: использование российских VPN-серверов**

Если VPN-сервер находится в России:

- Трафик до сервера — в России (бесплатно)
- Провайдер видит только трафик до российского IP
- Но VPN-сервер всё равно запрашивает контент за границей (сервер платит)

### Пример: сколько это будет стоить

**Сценарий: вы смотрите YouTube через VPN**

- **Потребление:** 2 часа видео 1080p ≈ 6 ГБ
- **День:** 3 часа видео ≈ 9 ГБ
- **Месяц:** 90 часов ≈ 270 ГБ

**Расходы:**

```
270 ГБ - 15 ГБ (бесплатно) = 255 ГБ
255 ГБ × 150 руб/ГБ = 38 250 руб/мес
```

Это дороже, чем платный VPN-сервис (обычно 300–1000 руб/мес).

**Вывод:** выгоднее использовать платный VPN, чем платить за каждый гигабайт.

## XRay VLESS перебои: МегаФон и другие провайдеры блокируют в выходные

### Хронология перебоев

**Пятница, 18 апреля:**

- Первые жалобы на Telegram-каналах VPN-сообществ
- МегаФон начинает блокировать Reality в некоторых регионах
- Пользователи сообщают о падении каждые 5–10 минут

**Суббота, 19 апреля:**

- Перебои становятся массовыми
- Жалобы от пользователей МегаФона, Билайна, МТС
- Reality работает 30–40% времени

**Воскресенье, 20 апреля:**

- Провайдеры активнее используют DPI
- Падения каждые 2–3 минуты
- Некоторым пользователям приходится отключать VPN

### Почему именно в выходные

**Гипотеза:**

1. **Нагрузка ниже** — в выходные пользователи реже работают, меньше корпоративного трафика
2. **Провайдеры тестируют** — внедряют новые правила DPI без риска сломать важный трафик
3. **Плановое обслуживание** — обновления оборудования ТСПУ

**Реальность:**

- В выходные провайдеры активнее режут VPN-трафик
- Пользователи жалуются, что приходится работать (IT-специалисты, фрилансеры)
- Нужен резервный протокол (например, Hysteria2)

### Что делать во время перебоев

**Краткосрочно:**

- Переключиться на другой провайдер (мобильный ↔ домашний)
- Попробовать другой протокол (Reality → Hysteria2)
- Использовать резервный конфиг с другим Server Name

**Долгосрочно:**

- Настроить **несколько протоколов** (Reality + XHTTP + Hysteria2)
- Использовать **резервные серверы** в разных странах
- Внедрить **автоматическое переключение** (failover)

- Боты используют VPN для масштабных атак

- ФСБ требует контроля над IT-компаниями

- Зашли на Ozon → отключили VPN → купили
- Зашли на Instagram → включили VPN → посмотрели сторис
- Зашли на ВТБ → отключили VPN → перевели деньги
- Зашли на YouTube → включили VPN → посмотрели видео

**Результат:** пользователям приходится включать/выключать VPN по 10–15 раз в день.

**Проблема 2: IT-специалисты теряют 30% времени**

Из статьи Habr "30% рабочего времени — борьба с DPI":

- IT-специалисты тратят 30% времени на настройку и отладку VPN
- Вместо разработки — борьба с блокировками
- Это прямой ущерб бизнесу

### Как обойти блокировку платформ

**Способ 1: split tunneling**

Настроить раздельную маршрутизацию:

- Российские сайты (Ozon, WB, Яндекс, банки) → напрямую (без VPN)
- Зарубежные сайты (YouTube, Instagram, Telegram) → через VPN

**Spécifity для разных клиентов:**

- **V2RayNG (Android)** — настроить правила маршрутизации
- **Streisand (iOS)** — использовать selective VPN
- **NEMO VPN** — автоматическая настройка правил

**Способ 2: residential IP**

Использовать VPN с residential IP вместо datacenter IP:

- Residential IP — адрес домашнего интернета
- Платформы реже блокируют residential IP
- NEMO VPN использует residential IP + VLESS Reality

**Способ 3: переключаться по необходимости**

Простой способ для не-IT пользователей:

1. Отключить VPN → зайти на Ozon/Яндекс → включить VPN
2. Использовать два профиля браузера (один с VPN, другой без)

## Что РАБОТАЕТ сейчас: VLESS Reality + XHTTP, Hysteria2, AmneziaWG

### VLESS Reality + XHTTP: следующая эволюция

**XHTTP** (Xray HTTP) — это мода на XRay, которая имитирует обычный HTTP/HTTPS трафик:

**Преимущества XHTTP:**

1. **Имитация HTTP** — выглядит как обычный браузерный трафик
2. **Меньше подозрений** — DPI видит HTTP запросы, а не неопознанный туннель
3. **Работает поверх Reality** — двойная обфускация

**Конфигурация XHTTP:**

```json
{
  "protocol": "vless",
  "settings": {
    "vnext": [{
      "address": "your-server.com",
      "port": 443,
      "users": [{
        "id": "your-uuid",
        "encryption": "none",
        "flow": "xhttp-rprx-vision"
      }]
    }]
  },
  "streamSettings": {
    "network": "xhttp",
    "security": "reality",
    "realitySettings": {
      "dest": "www.microsoft.com",
      "serverNames": ["www.microsoft.com"],
      "privateKey": "your-private-key",
      "shortIds": ["your-short-id"]
    },
    "xhttpSettings": {
      "path": "/random-path",
      "mode": "auto"
    }
  }
}
```

**Почему XHTTP важнее TLS:**

- TLS шифрует соединение, но DPI видит TLS handshake (JA3, SNI)
- XHTTP добавляет ещё один слой обфускации — имитацию HTTP запросов
- Это делает трафик менее подозрительным для DPI

**Hysteria2** — современный прокси-протокол на базе QUIC:

**Преимущества:**

**Почему Hysteria2 устойчив к DPI:**

- Нужен резервный протокол (Reality или XHTTP)

**Вывод:** Hysteria2 — мощный протокол, но не панацея. Используйте его в связке с Reality + XHTTP.

## Почему XHTTP важнее TLS: имитация обычного HTTPS-трафика

### TLS: защита, но не скрытие

**TLS (Transport Layer Security)** — это протокол шифрования:

- Защищает данные от перехвата
- Гарантирует подлинность сервера (сертификаты)
- Но НЕ скрывает факт зашифрованного соединения

**Что видит DPI:**

```
Client Hello (TLS 1.3) → DPI видит:
- Cipher suites
- Extensions
- JA3 fingerprint
- SNI (имя сервера)
```

Даже если вы используете TLS 1.3, DPI видит, что вы подключаетесь к VPN-серверу (по JA3 и SNI).

### XHTTP: имитация HTTP запросов

**XHTTP** (Xray HTTP) добавляет слой имитации HTTP поверх TLS:

**Что видит DPI:**

```
GET /random-path HTTP/1.1
Host: www.microsoft.com
User-Agent: Mozilla/5.0 ...
...
```

DPI видит:

1. **Нормальный HTTP запрос** на legit-домен (например, www.microsoft.com)
2. **TLS handshake** с легитимным сертификатом
3. **Ничего подозрительного** — это выглядит как обычный браузерный трафик

### Реальный пример: как XHTTP обходит DPI

**Сценарий: вы подключаетесь к VPN с XHTTP**

```
1. Ваше устройство → TLS handshake с www.microsoft.com (имитация)
2. DPI видит: обычный HTTPS-трафик на microsoft.com → пропускает
3. Внутри TLS-туннеля → XHTTP-запросы /random-path
4. XHTTP расшифровывается → настоящий VPN-трафик
5. VPN-сервер → запрашивает контент (например, YouTube)
```

**Без XHTTP:**

```
1. Ваше устройство → TLS handshake с vpn-server.com
2. DPI видит: JA3 fingerprint для XRay → подозрительно
3. DPI режет или замедляет трафик
```

### Почему XHTTP важнее TLS

| Характеристика | TLS | XHTTP |
|----------------|-----|-------|
| **Шифрование** | ✅ | ✅ (поверх TLS) |
| **Скрытие факта VPN** | ❌ | ✅ |
| **Детекция DPI** | Легко (по JA3) | Сложно (выглядит как HTTP) |
| **Скорость** | Высокая | Высокая (накладные расходы минимальны) |
| **Настройка** | Простая | Средняя (нужна конфигурация) |

**Вывод:** XHTTP критически важен для обхода DPI в 2026. TLS защищает данные, но XHTTP скрывает сам факт VPN.
