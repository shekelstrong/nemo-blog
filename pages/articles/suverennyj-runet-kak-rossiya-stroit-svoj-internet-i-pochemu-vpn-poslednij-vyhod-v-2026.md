## СОРМ vs ТСПУ: СОРМ копирует трафик спецслужбам, ТСПУ блокирует

Две ключевые системы контроля интернета в России — **СОРМ** и **ТСПУ** — выполняют разные функции.

### СОРМ (Система оперативно-розыскных мероприятий)

**Что делает:** копирует и передаёт трафик спецслужбам.

- **Не блокирует** — только собирает данные
- **Перехватывает** все коммуникации пользователя
- **Мониторит** телефонные звонки, SMS, интернет-трафик
- **Требует** от провайдеров установки оборудования для слежки

**Принцип работы:** провайдер устанавливает оборудование СОРМ, которое копирует весь трафик и передаёт в ФСБ, МВД и другие структуры. Пользователь об этом не узнаёт.

### ТСПУ (Технические средства противодействия угрозам)

**Что делает:** блокирует запрещённые ресурсы.

- **Блокирует** сайты, приложения, VPN
- **Использует DPI** (Deep Packet Inspection) для анализа пакетов
- **Работает на уровне** провайдеров и операторов связи
- **Контролируется** Роскомнадзором

**Принцип работы:** ТСПУ анализирует каждый пакет, проходящий через сеть, и блокирует, если он соответствует запрещённому критерию (IP-адрес, домен, протокол, ключевое слово).

### Ключевое отличие

| Характеристика | СОРМ | ТСПУ |
|---------------|------|------|
| Функция | Копирование трафика | Блокировка ресурсов |
| Контроль | ФСБ, МВД | Роскомнадзор |
| Влияние на пользователя | Невидимо | Блокирует доступ |
| Обход | Не нужен (не блокирует) | VPN, obfuscation |

**Важно:** СОРМ не блокирует — он только копирует и передаёт данные спецслужбам. ТСПУ блокирует доступ к ресурсам. Именно с ТСПУ борются VPN и obfuscation-инструменты.

## goodbyedpi и zapret: open-source обход без VPN

**goodbyedpi** и **zapret** — это open-source инструменты для обхода DPI-блокировок без использования VPN.

### goodbyedpi

**Что это:** утилита для обхода DPI на Windows и Linux.

**Принцип работы:**

- **Fragmentation:** разбивает TCP-пакеты на мелкие части
- **Host split:** разделяет заголовок Host на несколько пакетов
- **DNS redirect:** перенаправляет DNS-запросы
- **Fake SNI:** подменяет SNI (Server Name Indication)

**Установка на Windows:**

1. Скачайте [goodbyedpi](https://github.com/ValdikSS/goodbyedpi/releases)
2. Запустите с правами администратора
3. Выберите стратегию (recommended)
4. Перезагрузите браузер

**Установка на Linux:**

```bash
git clone https://github.com/ValdikSS/goodbyedpi.git
cd goodbyedpi
mkdir build && cd build
cmake ..
make
sudo ./goodbyedpi -s -f -p
```

### zapret

**Что это:** набор скриптов для обхода DPI-блокировок.

**Принцип работы:**

- **Fragment packets:** фрагментация пакетов
- **Host split:** разделение заголовка Host
- **DNS redirect:** перенаправление DNS-запросов

**Установка на Windows (Zapret GUI):**

1. Скачайте [Zapret GUI](https://github.com/bol-van/zapret-win-bat)
2. Запустите zapret-install-win7+.bat
3. Выберите стратегию (recommended)
4. Перезагрузите компьютер

**Установка на Linux:**

```bash
git clone https://github.com/bol-van/zapret.git
cd zapret
sudo ./install_easy.sh
```

### blockcheck.sh

**Что это:** скрипт для проверки стратегий обхода DPI.

**Как использовать:**

```bash

```

**Что проверяет:**

- Работают ли фрагментация пакетов

- Блокируется ли DNS
- Какая стратегия лучше всего

## DPI is all you need: как работает глубокая инспекция пакетов

**DPI (Deep Packet Inspection)** — это технология глубокого анализа сетевых пакетов.

**Как работает DPI:**

1. **Захват пакета:** DPI перехватывает каждый пакет, проходящий через сеть
2. **Анализ заголовков:** изучает IP-адрес, порт, протокол
3. **Инспекция payload:** проверяет содержимое пакета (SNI, TLS fingerprint)
4. **Принятие решения:** блокирует или пропускает пакет

**Что умеет DPI:**

- **Блокировать по IP:** заблокирован IP-адрес сайта — всё, к нему не пройти
- **Блокировать по домену:** SNI (Server Name Indication) показывает, к какому сайту вы подключаетесь
- **Блокировать по протоколу:** WireGuard, OpenVPN, Shadowsocks имеют уникальные характеристики
- **Блокировать по контенту:** проверяет содержимое пакетов на ключевые слова

**Блокировки на L3 уровне:**

- **L3 (Network Layer):** уровень IP-адресов
- **DPI может блокировать:** пакеты, направленные к определённым IP
- **Обход:** смена IP, прокси, VPN

**СОРМ vs DPI:**

| Характеристика | СОРМ | DPI |
|---------------|------|-----|
| Функция | Копирование трафика | Блокировка ресурсов |
| Уровень | L2-L7 | L3-L7 |
| Использование | Спецслужбы | РКН, провайдеры |
| Влияние | Невидимо | Блокирует доступ |

**Как DPI определяет VPN:**

1. **TLS fingerprinting:** каждая библиотека TLS имеет уникальный отпечаток (JA3, JA4)
2. **Протокол-специфичные характеристики:** WireGuard использует UDP с определённой структурой пакетов
3. **Порт-сканирование:** VPN-сервера часто используют нестандартные порты
4. **Активное зондирование:** DPI отправляет тестовые пакеты для проверки VPN-сервера

**Обход DPI:**

- **Obfuscation:** скрытие характеристик VPN-трафика (VLESS Reality)
- **Fragmentation:** разбивка пакетов (goodbyedpi, zapret)
- **Fake SNI:** подмена SNI (zapret)
- **Смена протокола:** использование протоколов, похожих на обычный HTTPS (XHTTP)

## Почему VPN всё ещё работает: шифрование скрывает содержание от ТСПУ

**VPN (Virtual Private Network)** создаёт зашифрованный туннель между вашим устройством и VPN-сервером.

**Почему VPN работает:**

1. **Шифрование:** весь трафик зашифрован, ТСПУ не видит содержимое
2. **Смена IP:** вы подключаетесь к VPN-серверу за пределами России
3. **SNI не раскрывается:** ТСПУ не видит, к какому сайту вы обращаетесь

**Но VPN можно определить:**

- **TLS fingerprinting:** Xray/VLESS имеет уникальный отпечаток
- **Активное зондирование:** DPI проверяет VPN-сервер
- **Анализ трафика:** паттерны VPN-трафика отличаются от обычного

**VLESS Reality — скрытие VPN-трафика:**

**VLESS Reality** — это протокол, который маскирует VPN-под HTTPS к реальному сайту.

**Как работает:**

1. **Server Name:** указываете реальный сайт (например, google.com)
2. **Dest:** реальный IP-адрес этого сайта
3. **Private Key:** генерируете ключ для подписи сертификатов
4. **VLESS Reality:** создаёт поддельный сертификат, который проходит проверку

**Результат:** ТСПУ думает, что вы подключаетесь к google.com, но на самом деле это VPN-сервер.

**NEMO VPN использует VLESS Reality:** весь трафик выглядит как обычный HTTPS к легитимным сайтам, ТСПУ не может его детектировать.
